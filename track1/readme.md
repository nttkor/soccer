[배경] 
현대 축구에서 승패는 개별 선수의 기량을 넘어, 보이지 않는 공간을 창출하고 유기적인 팀 패턴을 만드는 능력에서 갈립니다. 

그라운드 위 모든 움직임은 데이터로 기록되고 있지만, 이 방대한 데이터 속에서 실제 경기의 흐름을 바꾸는 ‘전술적 의도’를 읽어내는 것은 여전히 어려운 과제로 남아 있습니다.

기존의 축구 데이터 분석은 ‘패스한 선수와 공을 받은 선수’와 같은 단편적인 사실을 기록하는 데 그칩니다. 



하지만 진정한 통찰은 해당 시점의 선수 배치, 상대 압박, 공격의 전개 방향 등 복합적인 맥락 속에서 ‘왜 그 공간으로 패스하는 것이 최적이었는가’를 이해하는 데서 나옵니다.

이는 숙련된 선수처럼 경기 상황을 ‘이해’하고 다음 수를 예측해야 하는 고차원적인 문제입니다. 

이번 챌린지는 이러한 문제의식에서 출발합니다. K리그의 실제 경기 데이터를 기반으로, 단편적인 이벤트의 나열을 넘어 특정 상황의 맥락을 AI가 학습하고, 이어지는 패스가 도달할 최적의 위치를 예측하는 것을 목표로 합니다.

이를 통해 K리그 내 데이터 분석의 저변을 확대하고, 데이터 기반의 선수 평가 및 전술 분석에 대한 새로운 가능성을 발굴하고자 합니다.

>>Data Description

- train.csv: 학습 데이터
- test.csv: 테스트 데이터
- sample_submission.csv: 제출 양식
- submission.csv: 제출 파일
- submission.csv: 제출 파일

`train.csv`는 AI 모델을 가르치기 위한 **"교과서(정답지)"** 같은 데이터입니다.

이 파일 안에는 과거 실제 K리그 경기에서 발생한 **수많은 플레이 기록**이 들어있습니다.

### `train.csv`의 한 줄 한 줄의 의미
한 줄은 **"선수가 그라운드 위에서 한 행동 하나(Action)"**를 의미합니다.

주요 컬럼의 뜻은 이렇습니다:

*   **`game_id`**: 경기 번호 (어떤 경기인지)
*   **`episode_id`**: 공격 시퀀스 번호 (한 번의 공격 기회)
*   **`start_x`, `start_y`**: 행동이 **시작된 위치** (예: 패스를 찬 곳)
*   **`end_x`, `end_y`**: 행동이 **끝난 위치** (예: 공이 도착한 곳)
    *   **★우리의 목표★**: 나중에 AI가 `start_x, y`만 보고 이 `end_x, y`를 맞추게 하는 것입니다.
*   **`type_name`**: 행동 종류 (Pass, Dribble, Shot 등)
*   **`result_name`**: 결과 (성공했는지 실패했는지)
*   **`player_id`**: 어떤 선수가 했는지

---

### 요약하자면
**"이 상황(`start_x, y`)에서 이 선수가 찼더니 공이 저기(`end_x, y`)로 갔다"** 라는 기록들의 모음집입니다.

우리는 이 데이터를 모델에게 보여주면서,
**"봐봐, 여기서 차면 저기로 가지? 이걸 수십만 번 보고 패턴을 익혀!"** 라고 훈련시키는 것입니다.

네, 맞습니다! 구조가 조금 특이해서 헷갈리실 수 있습니다.

### 1. `train.csv` (단일 파일)
- **학습용 데이터 전체**가 이 파일 하나에 다 들어있습니다.
- 약 35만 줄의 데이터가 있고, 각 행은 패스나 드리블 같은 액션 하나하나입니다.

### 2. `test` 데이터 (파일 + 폴더 구조)
테스트 데이터는 **"문제지"**인데, 두 부분으로 나뉘어 있습니다.

#### (1) `test.csv` (목차 파일)
- **"어떤 문제를 풀어야 하는지"** 목록이 적혀 있습니다.
- 컬럼: `game_id`, `game_episode`, **`path`** (데이터 파일 위치)
- 예시: "153363번 경기의 1번 에피소드 문제는 `path`에 있는 파일을 봐라"

#### (2) `test/` 폴더 (실제 문제 내용)
- `test.csv`의 `path`가 가리키는 **실제 csv 파일들**이 들어있는 폴더입니다.
- 예: `test/153363/153363_1.csv`
- **각 파일 하나가 하나의 문제(에피소드)**입니다.
- 이 파일 안에는 그 에피소드에서 일어난 패스, 드리블 등의 과정이 들어있습니다.
- **마지막 줄**의 `end_x`, `end_y`는 비어있거나 우리가 맞춰야 할 미래입니다.

### 정리
- **학습할 때**: `train.csv` 하나만 주구장창 파면 됩니다.
- **시험 볼 때**: `test.csv`를 읽어서 목록을 확인하고 -> 각 목록에 적힌 `test/폴더/파일.csv`를 하나씩 열어서 문제를 풀어야 합니다.
